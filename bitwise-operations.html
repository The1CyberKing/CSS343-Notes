Bitwise Operations

What are Bitwise Operations?

  • Bitwise operations are operations that directly manipulate individual bits (0s and 1s) of binary numbers using logical operators like AND, OR, XOR, NOT, and bit shifts.

Why use them?

  • If you write in C++, there's a great chance you will need to use it sooner or later.
  • Most popular languages support bitwise operations
  • Bitwise operators are best aligned with when you use C/C++ since the languages are usually used when you need to do fine-grained bit manipulation

Bits, Bytes, and their Definitions

  • Base Meaning: The number of unique digits a system uses (EX: Base 10 uses 0–9)
  • Decimal Number Meaning: Normal number: base 10 EX: 1785
  • Binary Number Meaning: Base 2, uses 0 and 1 EX: 11011111001
  • Octal Number Meaning: Base 8, uses 0–7 EX: 3361
  • Hexadecimal Number Meaning: _Base 16, uses 0–9 and A–F EX: 6F9

Notation

  • Decimal: No Prefix; EX: 1785
  • Binary: Prefix: 0b or 0B; EX: 0b10110
  • Octal: Prefix: Leading 0; EX: 0361 Valid pre-C++14, or rather discouraged in newer versions
  • Hex: Prefix: 0x or 0X; EX: 0x6F9

Conversions

NOTE: you could just convert everything to binary and then convert to whatever base you'd like.

Decimal Number

Decimal → Binary

Rule: Repeatedly divide by 2 and write down the remainders (bottom to top).
Example

13 ÷ 2 = 6 remainder 1

6 ÷ 2 = 3 remainder 0

3 ÷ 2 = 1 remainder 1

1 ÷ 2 = 0 remainder 1

Result: *1101*

Decimal → Octal

Rule: Divide by 8 and write remainders (bottom to top).
Example

83 ÷ 8 = 10 remainder 3

10 ÷ 8 = 1 remainder 2

1 ÷ 8 = 0 remainder 1

Result: *123*

Decimal → Hex

Rule: Divide by 16 and use letters A–F for 10–15.
Example

254 ÷ 16 = 15 remainder 14

15 = F, 14 = E

Result: *FE*

Binary Number

Binary → Decimal

Rule: Multiply each bit by 2 raised to its position power, then add them all together.
Example

(1 × 2⁴) + (0 × 2³) + (1 × 2²) + (1 × 2¹) + (0 × 2⁰)

= 16 + 0 + 4 + 2 + 0

= 22₁₀

Result: 22

Binary → Octal

Rule: Group bits into sets of 3 from right to left, then convert each group to its decimal value.
Example

Group bits into 3s from right: 110 101

110 → 6

101 → 5

Result: 65

Binary → Hex

Rule: Group bits into sets of 4 from right to left, then convert each group to its hexadecimal value.
Example

Group bits into 4s from right: 1110 1110

1110 → E

1110 → E

Result: EE

Octal Number Not Discussed in class

Octal → Decimal

Rule: Multiply each digit by 8 raised to its position power, then add them all together.
Example

347₈ = (3×8²) + (4×8¹) + (7×8⁰) = 231₁₀

Result: 231

Octal → Binary

Rule: Replace each octal digit with its 3-bit binary equivalent.
Example

347₈ → 3 4 7 → 011 100 111 → 011100111₂

Result: 011100111

Octal → Hex

Rule: First convert octal to binary, then group the binary digits into 4s to get hexadecimal.
Example:

347₈ → 011100111₂ → 0001 1100 1111 → 1CF₁₆

Result: 1CF

Hexadecimal Number

Hex → Decimal

Rule: Multiply each digit by 16 raised to its position power, then add them all together.
Example

2F₁₆ = (2×16¹) + (15×16⁰) = 47₁₀

Result: 47

Hex → Binary

Rule: Replace each hex digit with its 4-bit binary equivalent.
Example

2F₁₆ → 2 F → 0010 1111 → 00101111₂

Result 00101111

Hex → Octal

Rule: Convert hex to binary, then group the binary digits into 3s from right to left to get octal.
Example

2F₁₆ → 00101111₂ → 000 010 111 1 → 057₈

Result: 057

Bitwise AND (&)

  • Compare each bit at the same position in the integer
  • If both are 1, then the result is 1. Any other case doesn't result in this (0).

Example

unsigned result = 7&4; //Equals 4

  1 1 1
& 1 0 0
---------
  1 0 0

7 is 111, and 4 is 100

Bitwise OR (|)

  • Compare each bit at the same position in the integer
  • if either of them is 1, then the result is 1. if both are 0, the result is 0.

Example

unsigned result = 7|4; //Equals 7

  1 1 1
| 1 0 0
---------
  1 1 1

7 is 111, and 4 is 100

Bitwise XOR (^)

  • Compare each bit at the same position in the integer
  • if the bits are the same, it will be 0. if they are different, it is 1.
  • It's a reversable process (The reason it's used often!) A ^ B ^ B = A

Example

unsigned result = 7^4; //Equals 3

  1 1 1
^ 1 0 0
---------
  0 1 1

7 is 111, and 4 is 100

Bitwise NOT (~)

  • Preformed on a single number.
  • Will essentially make any 1 to a 0, and 0 to a 1. it's "complement"

Example

unsigned result = ~4; //Equals 3 Be aware of bit length, because this final value can be very different. this is only true for 3 bits. C++ doesn't choose 3 bit as default.

~ 1 0 0
---------
  0 1 1

4 is 100

Left Shift (<<)

  • Shifts bits of an integer to the left by a specific number of positions.
  • Same thing as #

Example

unsigned result = 5<<2; //Equals 20

0 0 0 0 0 1 0 1 << 2
---------------
0 0 0 1 0 1 0 0

Right Shift (>>)

  • Shifts bits of an integer to the right by a specific number of positions.
  • Same thing as #

Example

unsigned result = 16>>2; //Equals 4

0 0 0 1 0 0 0 0 >> 2
---------------
0 0 0 0 0 1 0 0

About Shifting

  • In C++ shifting by 0 bytes doesn't do anything
  • Don't shift by equal or more than the available bit length, it's undefined behavior.
  • You can make constraints to help with the previous point.

Little Endian vs Big Endian

  • Endianness describes how multi-byte values are laid out in memory.

Little Endian

  • The Least Significant Byte is stored at the lowest memory address.
  • This is used by Intel (x86 & x64) CPUs. And by most modern computers.
    Example
uint32_t x = 0xA1B2C3D4;

Memory Storage (low high)

D4 C3 B2 A1

Big Endian

  • The Most Significant Byte is stored at the lowest memory address.
  • Used by network protocols, and similar systems.
    Memory Storage (low high)
A1 B2 C3 D4

Why do these exist?

  • There must be a predetermined format for different processes to read the bytes. Especially when sending data over network.
  • NOTE: C/C++ provides conversion helpers to convert between the two.

Flags

  • Flags are individual bits packed into an integer, where each bit represents an on/off option.
  • Extremely common in system calls, file I/O, graphics APIs, embedded systems, and OS kernels.
  • Using flags avoids passing many separate boolean parameters.

Why flags are useful

  • Compact (32 or 64 options in a single integer)
  • Fast & cheap on hardware
  • Easily to create

Defining flags

namespace OpenFlags {
constexpr uint32_t READ = 1 << 0; // 0001
constexpr uint32_t WRITE = 1 << 1; // 0010
constexpr uint32_t APPEND = 1 << 2; // 0100
constexpr uint32_t TRUNC = 1 << 3; // 1000
}

Bit sets

  • A bitset is a collection of bits where each bit is like a boolean value.
  • Similar to a boolean array, but way more memory efficient.

Manual bitsets (bit manipulation)

  • Each bit position represents an index or ID
  • Stored using integers or arrays of bytes
    Example
    friend list encoded in a uint16_t
// bit positions: [15 ... 0]
uint16_t recipients = (1 << 0) | (1 << 4) | (1 << 7);

Why use it?

  • Very compact
  • CPU Cache-friendly
  • Ideal for large datasets or tight memory constraints

STL std::bitset

#include <bitset>
std::bitset<128> users;
users.set(5);
users.reset(12);
users.flip(7);
  • Size fixed at compile time
  • Bounds-checked access (safer than manual bit math)
  • Very fast

Boolean Variables

  • Keeping many boolean values as bool is memory-inefficient.
  • A bool usually uses 1 byte, even though it only needs 1 bit.

Example comparison

  • 50 boolean values
    • bool flags[50] → 50 bytes
    • Bitfield → 50 bits (~7 bytes)

Efficient representation

uint64_t features = 0;
features |= (1ULL << 3); // enable feature 3
features |= (1ULL << 10); // enable feature 10

Checking a boolean value

bool enabled = features & (1ULL << 10);

When to use bitfields

  • Large datasets
  • Performance-critical code (For systems with low power microcomputers) Like when coding an STM microcontroller!
  • Memory-constrained systems

Compression

  • Compression often uses bit packing (like Huffman coding)

Hashing

  • Hash functions use XOR and shifts for mixing bits
  • Used when creating as unique a value as possible
