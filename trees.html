Trees

Why Use Trees?

  1. They are very good for adding and deleting
  2. Quite a few applications, which include:
    • Huffman Tree for Comparison
    • Ternary Trees for text search
    • Red/Black Trees for language parsing
    • Literally anything Hierarchical

Pre-Order - Root Left Subtree Right Subtree

Definition

  • Visit the root first, then recursively traverse the left subtree, then the right subtree.
    Phrase
  • PREpare My Little Rat (Pre: Me Left Right)
    Example
  • 1,2,4,5,3,6,7

In-Order - Left Subtree Root Right Subtree

Definition

  • Recursively traverse the left subtree, visit the root, then traverse the right subtree.
    Phrase
  • IN Leaving Mail Responsibly (In: Left Me Right)
    Example
  • 4,2,5,1,6,3,7

Post-Order - Left Subtree Right Subtree Root

Definition

  • Recursively traverse the left subtree, then the right subtree, and visit the root last.
    Phrase
  • POST Letters Really Meticulously (Post: Left Right Me)
    Example
  • 4,5,2,6,7,3,1

Vocabulary

Height/Depth

  • Number of Nodes between the root and a leaf.
  • The Root has a height of 0.
    • This is how Professor Reel Defined it.
    • This is not fully conventional as some courses say the leaves have height 0 instead.

Level

  • Number of nodes necessary to make it to a selected node from the root

Balanced

  • The difference of heights of left and right subtrees at any node is .

Complete

  • It is complete when All levels of the tree are filled completely except for the lowest level nodes which are filled as much from the left as possible.

Full

  • A binary tree where each node has either 0 or 2 children.

Degenerate

  • Each parent node has only one child node.

Huffman Coding

What is it?
  • A Common Form of Compression
  • It assigns variable length codes to symbols based on their frequency of occurrence (More frequent ones get shorter codes)
  • 30-50% compression rate (for raw text)
  • Lossless!
  • Used in common things like ZIP, PNG, MP3, JPEG!
How does it work?
  • Build a binary tree starting with nodes that appear less frequently
  • assign 0 and 1 to the edge of it's left and right children
  • follow the edges from the root to find the Huffman code
  • makes sure no piece of data will have the same encoding as another piece of data
Actual Steps (from lecture slides)
  1. Create a leaf node for each unique character in the input.  The node should contain the character and its frequency.
  2. Dump all of the nodes into a priority queue (min-heap).
  3. Extract the two nodes with the lowest frequencies from the queue
  4. Create a new internal node with a frequency equal to the sum of the two nodes.  Make the first extracted node the left child and the other the right child.  Add the internal node to the queue.
  5. Repeat steps 3 & 4 until the queue contains only one node.
